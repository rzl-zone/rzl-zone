## Supported Versions

Only the latest stable release of **Build Tools** will receive security updates.  
Please make sure you are using the most recent version before reporting an issue.

<SeparatorSection />

## Reporting a Vulnerability

If you discover a security vulnerability, please report it by contacting:

- 📧 Email: rzlzone.dev@gmail.com.
- 🔒 Private disclosure (recommended): [Report a vulnerability securely.](https://github.com/rzl-zone/rzl-zone/security/advisories/new)
- 🐛 Public issue (if necessary): [Report security-related issue.](https://github.com/rzl-zone/rzl-zone/issues/new?labels=build-tools,security&title=[build-tools][security]%20&body=Package:%20@rzl-zone/build-tools%0A%0AType:%20Security%20Issue%0A%0ADescription:%0A%0A-%20Steps%20to%20reproduce:%0A-%20Impact:%0A-%20Expected%20behavior:%0A-%20Actual%20behavior:%0A-%20Environment:%0A)  


We will review and respond to your report as quickly as possible.  
Please **do not disclose the vulnerability publicly** until it has been addressed.

<SeparatorSection />

## Responsible Disclosure

To protect the community, we kindly ask that you:

1. Provide detailed steps to reproduce the vulnerability.
2. Allow reasonable time for us to investigate and patch before any public disclosure.
3. Avoid exploiting vulnerabilities for any malicious purpose.

<SeparatorSection />

## Scope 

- ✅ Vulnerabilities affecting the core logic, security boundaries, or data integrity of **Build Tools**.
- ❌ Non-security bugs (please use the [issue tracker](https://github.com/rzl-zone/rzl-zone/issues?q=is%3Aissue+label%3Abuild-tools) for those). 

<SeparatorSection />

🙏 Thank you for helping keep **Build Tools** and its users safe!
